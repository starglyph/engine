## Why

После завершения dataset v1 проект умеет генерировать воспроизводимую синтетику, но пока не умеет решать основную пользовательскую задачу: распознавать участок неба на изображении. Фаза 2 нужна сейчас, чтобы построить минимальный solver v1, который замкнет цикл "кадр -> идентификация -> оверлей" на clean synthetic данных и задаст измеримую базу для следующих фаз.

## What Changes

- Добавляется baseline-пайплайн детекции звезд на synthetic кадрах с оценкой precision/recall против truth-данных.
- Добавляется базовый matching звездных паттернов (треугольники/квады или эквивалент) с confidence-оценкой и фильтрацией неоднозначных кандидатов.
- Добавляется оценка ориентации камеры по сопоставленным точкам с робастной фильтрацией выбросов.
- Добавляется рендер оверлея (линии созвездий) и отладочные слои (детекции, inliers/outliers).
- Добавляется benchmark harness для пакетного прогона solver по synthetic dataset с отчетом метрик и автосохранением худших кейсов.

## Capabilities

### New Capabilities

- `solver-star-detection`: детекция звездных кандидатов на synthetic кадре с вычислением и отчетом метрик качества.
- `solver-pattern-matching`: сопоставление детекций с каталогом звездных паттернов и вычисление confidence для гипотез.
- `solver-pose-estimation`: восстановление ориентации камеры по сопоставленным звездам с устойчивостью к выбросам.
- `solver-overlay-debug`: визуализация распознанного участка неба и debug-слоев поверх входного изображения.
- `solver-benchmark-harness`: пакетный запуск solver по датасету с агрегированными метриками и сохранением failure-cases.

### Modified Capabilities

_None._

## Impact

- Затрагиваются будущие модули распознавателя (детекция, matching, pose, overlay, benchmark) и интерфейсы обмена с synthetic dataset v1.
- Понадобятся новые спецификации в `openspec/changes/roadmap-phase-2-solver-v1/specs/` для всех capability фазы 2.
- Метрики фазы (precision/recall, угловая ошибка pose, воспроизводимость benchmark) становятся формальным критерием готовности solver v1.
