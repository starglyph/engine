# Стек прототипа (десктоп)

Зафиксированное решение для **первого прототипа**: нативное **десктоп-приложение под Linux** (в т.ч. **Linux Mint**), одно окно с **двумя панелями**: слева — кадр **симулятора**, справа — результат **распознавателя** (детекция, matching, оверлей и диагностика).

## Язык и рантайм

- **Rust** — основной язык для прототипа: симулятор, обработка изображений, геометрия небесной сферы, сопоставление с каталогом и отрисовка оверлея в одном процессе, предсказуемая производительность, удобный деплой одним бинарником на Mint.

## Пользовательский интерфейс

Приоритет — **быстрый внутренний прототип** с двумя областями изображения и элементами управления параметрами симуляции.

| Вариант | Когда уместен |
|---------|----------------|
| **`egui` + `eframe`** | Минимум церемоний, immediate mode, быстро собрать два «холста», слайдеры, логи. Рекомендуемая стартовая точка. |
| **`iced`** | Если UI быстро разрастаётся и нужен более явный поток состояния (MVU). |

**GTK 4 (`gtk4-rs`)** или **Qt** — резерв, если позже понадобится нативный вид системы или долгоживущий сложный UI; для первой версии с двумя панелями обычно избыточно.

## Зависимости и экосистема (ориентиры, не жёсткий список)

- **Линейная алгебра / векторы:** `glam`, `nalgebra` — по выбранным соглашениям в коде.
- **Изображения:** `image` и/или прямой доступ к буферу для отрисовки в UI.
- **Каталоги и форматы:** отдельные модули + документированные артефакты (см. `simulator.md`); лицензии каталогов учитывать до подключения данных.

## Что сознательно не фиксируем на старте

- Конкретные версии крейтов и workspace-структура — появятся с первым коммитом кода.
- Продвинутый ML в Rust: при необходимости тяжёлые эксперименты можно вести **вне** этого приложения (например, Python + обучение), а в десктоп протаскивать уже **выгруженную модель** (ONNX и т.п.) — отдельным решением по мере появления требований.

## Связь с дорожной картой

Соответствует ранним фазам в [roadmap.md](roadmap.md): симулятор → распознаватель на синтетике → затем усложнение реализма и реальные фото. Мобильный клиент остаётся **перспективой** и не входит в стек текущего прототипа.
