# Имитатор звёздного неба (часть 1)

## Цель

Получать изображения неба с **известной геометрией и составом**: какие звёзды попали в кадр, с какими координатами на сенсоре и видимыми величинами. Это основа для обучения, отладки распознавателя и регрессионных тестов.

## Входные данные симуляции

1. **Каталог** — минимум: экваториальные координаты (α, δ), видимая звёздная величина; опционально: собственные движения, двойные звёзды, цвет (B−V).
2. **Момент времени** — для аберрации, прецессии (если моделируем строго), положения планет.
3. **Наблюдатель** — широта/долгота или локальная система; для «чистого» неба можно задать напрямую **направление оптической оси** и **поворот вокруг оси** (roll).
4. **Камера** — поле зрения (или фокусное + размер сенсора), разрешение, модель дисторсии (pinhole + radial/tangential), кривая отклика по яркости (опционально).

## Проекция на плоскость изображения

Типичный путь: экваториальные координаты → **международные небесные координаты** (ICRS) в нужный эпохерис → преобразование в **горизонтальную** или сразу в **систему камеры** (вектор на небесной сфере) → **проекция** (равнопромежуточная, стереографическая, «рыбий глаз» и т.д.) → пиксели.

Для синтетики важно хранить не только RGB, но и **метаданные кадра**: матрица внешней ориентации, внутренние параметры, список видимых звёзд с пиксельными координатами и m_v.

## Реализм изображения

Слои, которые можно включать поэтапно:

| Слой | Описание |
|------|----------|
| PSF звезды | Гауссиан / эйри-диск + хвосты от оптики |
| Отклик сенсора | shot noise, read noise, квантование, bloom на ярких |
| Атмосфера | seeing (случайное «дрожание»), рассеяние от Луны/города (фон) |
| Оптика | виньетирование, хроматизм, блики |
| Артефакты | горячие пиксели, полосы, сжатие JPEG (если имитируем телефон) |

Чем ближе симулятор к реальности, тем **меньше domain gap** для нейросетей и тем **сложнее** поддерживать симулятор.

## Выход артефактов

Рекомендуемый минимум для пайплайна обучения:

- `image.png` или аналог
- `meta.json` — время, камера, ориентация, версия каталога
- `stars.csv` или бинарь — id, x_px, y_px, flux или m_v

Формат можно зафиксировать отдельно, когда появится код.

## Риски и ограничения

- **Лицензии каталогов** — проверить условия использования (особенно для коммерческого приложения).
- **Слишком идеальные** кадры — распознаватель переобучится; нужен баланс шума и искажений.
- **Планеты** — яркие диски/фазы; для симулятора понадобится эфемериды (например, через готовую библиотеку).

## Возможные этапы реализации

1. Идеальная pinhole-камера, без шума — проверка астрометрии.
2. Шум + PSF — первые «реалистичные» кадры.
3. Дисторсия и виньетирование — ближе к телефону/широкоугольнику.
4. Пакетная генерация датасета с вариацией параметров.
