# Источники фото звёздного неба, лицензии и согласие на обработку

Документ описывает, **откуда брать реальные снимки** для разработки и приёмки
распознавателя, какие с этим связаны **юридические риски** и как их закрывать в
условиях **публичного репозитория** (код — Apache-2.0, данные лицензируются
отдельно; см. [../THIRD_PARTY_LICENSES.md](../THIRD_PARTY_LICENSES.md),
[../ATTRIBUTION.md](../ATTRIBUTION.md)).

Про звёздные **каталоги** (HYG, d3-celestial) — не здесь, а в
[phase0.md §0.3](phase0.md) и `THIRD_PARTY_LICENSES.md`. Здесь речь о
**фотографиях неба** (пиксельные кадры для детекции/приёмки).

## 1. Стратегия данных (три уровня)

Проект синтетический по замыслу, поэтому реальные фото — не основной обучающий
корпус, а средство проверки на настоящем шуме и оптике.

| Уровень | Назначение | Юр. риск | Ground truth |
| ------- | ---------- | -------- | ------------ |
| **Синтетика** (фазы 1–2 дорожной карты) | Основной объём обучения/бенчмарков | Нет | Полный, точный (RA/Dec, FOV, поза) |
| **Открыто-лицензированные архивы** | Валидация на реальном небе | Низкий при соблюдении лицензии | Часто нет — нужно решать самим |
| **Собственная/донорская съёмка** | «Золотой» тест реального железа | Низкий при оформлении прав/согласия | Метаданные съёмки как near-truth |

Практический нюанс для blind plate solving: ценнее всего кадры с **известной
привязкой** (куда смотрела камера, FOV) — иначе «курица и яйцо». Синтетика даёт
это бесплатно; у реальных фото привязку либо берём из метаданных, либо получаем
самим решателем и фиксируем как эталон после ручной проверки.

## 2. Безопасные источники в интернете

Брать только материалы с **явной свободной лицензией** (CC0 / CC BY / CC BY-SA /
Public Domain). Лицензия проверяется **на странице конкретного файла** — она
бывает per-image и меняется.

| Источник | Лицензия (типовая) | Замечание |
| -------- | ------------------ | --------- |
| **Astrometry.net** (nova.astrometry.net, галерея/тестовые наборы) | смешанно, часть CC | Реальные любительские кадры с **уже решённой привязкой** — идеально для приёмки; лицензию смотреть у каждого кадра |
| **ESO** (eso.org/public/images) | CC BY 4.0 | Обязательна атрибуция |
| **ESA/Hubble, ESA/Webb** | CC BY 4.0 | Атрибуция |
| **NASA** (images.nasa.gov) | в основном Public Domain | Встречаются вкрапления чужих ©, читать подпись под каждым файлом |
| **NOIRLab** (noirlab.edu) | CC BY 4.0 | Атрибуция |
| **Wikimedia Commons / Flickr** | фильтр по CC BY / CC0 | Фильтровать по лицензии; сохранять автора для атрибуции |

**Избегать без явного разрешения автора:** AstroBin, астрофорумы, случайные
картинки из поиска — по умолчанию всё под полным © (Бернская конвенция), даже
любительское и «просто звёзды».

Любой заимствованный кадр, попадающий в репозиторий, **обязан** быть внесён в
реестр §5 и в `ATTRIBUTION.md` до коммита.

## 3. Юридические риски (по значимости)

1. **Авторское право.** Фото защищено © по умолчанию. Массовое скачивание
   произвольных снимков в датасет — риск нарушения; «для исследований» ≠
   автоматически законно. Митигация: только CC/PD + реестр лицензий и атрибуций.
2. **Персональные данные (GDPR / 152-ФЗ) — самый недооценённый пункт.** Риск не
   в звёздах, а в **EXIF**: GPS-координаты, точное время, серийник/модель
   устройства, имя владельца; плюс люди/номера/таблички в кадре переднего плана.
   Митигация: стрип EXIF на входе (оставлять только съёмочные поля, если нужны),
   осознанное согласие, если геометки сохраняются специально как ground truth.
3. **Sui generis database right (ЕС).** Касается баз/каталогов, не отдельных фото;
   для HYG/Gaia не проблема (распространяются для свободного использования), но
   знать о нём стоит.
4. **Смешение лицензий в публичном репо.** Файл без явной лицензии рядом с
   Apache-2.0-кодом читается как «все права защищены» и блокирует легальное
   переиспользование клонирующими. Любой кадр в репо должен иметь явно указанную
   лицензию (см. §4, §5).

## 4. Собственная и донорская съёмка

### 4.1 Личные снимки автора

Кадры автора проекта (текущее содержимое `data/input/`, серия 2011-09-20 и др.) —
правообладатель известен, риск нарушения чужих прав отсутствует. Для публичного
репозитория остаётся **явно указать лицензию** на эти данные, иначе они
де-факто «все права защищены».

Рекомендуемое действие: выбрать лицензию данных (например, **CC BY 4.0** —
простое переиспользование с атрибуцией) и зафиксировать её в
`data/input/README.md` и в реестре §5. Решение о лицензии — за автором.

### 4.2 Съёмка у знакомых (доноров)

Рабочий и «чистый» путь для расширения набора реальным железом, если оформить:

- **Письменное согласие** донора (шаблон — Приложение A): цель обработки, факт и
  форма публикации, лицензия передачи.
- **Явная лицензия** передаваемых кадров (проще всего — CC BY 4.0 или прямая
  передача проекту без ограничений).
- **Стрип EXIF на входе**: убирать GPS/серийники/имя; при необходимости оставлять
  только съёмочные поля (фокусное, выдержка, ISO). Если геометки нужны как
  ground truth — это осознанно обрабатываемые ПДн, и согласие обязано это
  покрывать.
- **Бонус:** попросить параметры съёмки (объектив, направление, время) — половина
  ground truth бесплатно.

**Заполненные согласия — это ПДн доноров и в публичный git не коммитятся.**
Хранить приватно (вне репо / защищённое хранилище); в репозитории — только факт
наличия согласия и лицензия кадра в реестре §5.

## 5. Реестр реальных фото (провенанс + лицензия)

Обязателен для каждого реального кадра, попадающего в репозиторий. Ведётся здесь
и/или в `ATTRIBUTION.md`; фактические файлы данных — по правилам `.gitignore`.

| Набор / файлы | Источник | Правообладатель | Лицензия | Согласие | Ground truth |
| ------------- | -------- | --------------- | -------- | -------- | ------------ |
| `data/input/*.bmp` (серия 2011-09-20 и др.) | Собственная съёмка | Igor Lazarev | **CC BY 4.0** | н/п (автор) | параметры в имени файла; привязка — по решателю. Детали: [`../data/input/README.md`](../data/input/README.md) |

> Новые источники добавляются строками сюда **до** коммита данных.

## 6. Рекомендация для starglyph

- **Основа — синтетика** (0 рисков, полный ground truth).
- **Реальная валидация** — Astrometry.net (решённые кадры) + ESO/ESA/NASA под
  CC/PD, с реестром §5 и атрибуцией.
- **Личные/донорские кадры** — «золотой» тест реального железа: явная лицензия,
  для доноров — согласие + стрип EXIF.
- **Данные ≠ git-история:** тяжёлые/чужие материалы держать по `.gitignore` с
  манифестом лицензий, чтобы не зашить чужой © в публичную историю.
