# Обзор и цели

## Идея

Сделать конвейер «**синтетическое небо → распознавание → визуализация**», который в итоге может лечь в основу приложения: по фото участка неба определить, **что за регион неба** на кадре, **дорисовать созвездия** и отметить **интересные объекты** (планеты, яркие объекты глубокого космоса — по мере развития).

Отдельно перспектива — **мобильное приложение**: камера в реальном времени, те же алгоритмы с учётом ограничений по вычислениям и задержке.

## Две части проекта

### 1. Имитатор звёздного неба

Генерация снимков, максимально похожих на реальные: звёздное поле, блеск, шум камеры, оптические артефакты, атмосфера (по желанию), разные выдержки и поля зрения.

**Зачем:** контролируемая «истина» (какие звёзды где и с какой яркостью), масштабируемый **датасет** для обучения и тестов, воспроизводимые бенчмарки.

### 2. Распознаватель

По изображению (реальному или синтетическому) восстановить **положение камеры относительно небесной сферы** (или эквивалент: направление взгляда + поле зрения), сопоставить с **астрономическим каталогом**, затем нарисовать поверх кадра **линии созвездий**, подписи, планеты и т.д.

## Конечный пользовательский сценарий

1. Пользователь загружает фото ночного неба (или снимает камерой).
2. Система определяет ориентацию и область неба.
3. На изображение накладываются созвездия и выбранные объекты в корректной геометрии.
4. В перспективе — интерактив: движение телефона, обновление оверлея.

## Ключевые технические вопросы (для последующей детализации)

- **Каталог звёзд:** Hipparcos, Tycho-2, часть Gaia — компромисс точность / размер / лицензия.
- **Модель камеры:** внутренние параметры (фокусное, дисторсия), внешняя ориентация (матрица или кватернион + углы).
- **Matching:** классические методы (triangle matching, ASIFT-like features) vs обучаемые детекторы ключевых точек + pose estimation.
- **Созвездия:** границы IAU, линии между звёздами — отдельный слой данных поверх каталога.
