# Фаза 0 — scope, данные и контракты

Документ фиксирует решения по пунктам `0.1–0.5` из [roadmap.md](roadmap.md).

## 0.1 Сценарии использования

### Обязательный режим (MVP)

- Вход: одна фотография ночного неба (`jpg/png`), без обязательных EXIF-полей.
- Выход:
  - оценка позы камеры (направление взгляда, roll, FOV),
  - оверлей линий созвездий на исходном кадре,
  - диагностический отчёт по уверенности сопоставления.

### Расширенный режим

- Дополнительный вход: дата/время и геолокация:
  - автоматически из EXIF, если поля доступны;
  - вручную, если EXIF отсутствует или недостоверен.
- Эффект: сокращение пространства поиска при matching и доступность слоёв, зависящих от времени (планеты, яркие объекты по эфемеридам).

## 0.2 Критерии качества

Фиксируем baseline-метрики и пороги для двух уровней: `MVP` и `Next` (после стабилизации на synthetic + real eval).


| Метрика                                           | MVP                 | Next                |
| ------------------------------------------------- | ------------------- | ------------------- |
| Угловая ошибка оси камеры (median / p95)          | `<= 0.8° / <= 1.5°` | `<= 0.3° / <= 0.7°` |
| Ошибка roll (median / p95)                        | `<= 1.5° / <= 3.0°` | `<= 0.7° / <= 1.5°` |
| Precision детекции звёзд (synthetic clean)        | `>= 0.90`           | `>= 0.95`           |
| Recall детекции звёзд (synthetic clean)           | `>= 0.88`           | `>= 0.93`           |
| Время обработки кадра 12 MP на CPU (median / p95) | `<= 2.0s / <= 3.0s` | `<= 0.7s / <= 1.2s` |


Методика расчёта и состав отчёта определены в [evaluation.md](evaluation.md).

| Набор                                    | Назначение                                        | Лицензия       | Примечание                                                                       |

Принятое соглашение по именам:

- Внутренний ключ: 3-буквенное сокращение IAU (`Ori`, `UMa`, ...).
- Человекочитаемое имя: английское название IAU.
- Локализация (например, RU) добавляется отдельным словарём, не меняет стабильный ключ.

Юридический статус для фазы `0`:

- Использование в прототипе допустимо при сохранении атрибуции и текста лицензий в репозитории.

- Для репозитория добавлены:
  - [`../THIRD_PARTY_LICENSES.md`](../THIRD_PARTY_LICENSES.md) — реестр лицензий и обязательств;
  - [`../ATTRIBUTION.md`](../ATTRIBUTION.md) — готовые тексты атрибуции и release-чеклист.

## 0.4 Форматы артефактов

Форматы synthetic-пакета, каталогов и версионирования зафиксированы в [data-contracts.md](data-contracts.md).

Коротко:

- Единица набора: `frame package` (изображение + параметры камеры + truth).
- Разделение по подсистемам:
  - `artifacts/simulator/...` — данные генератора;
  - `artifacts/recognizer/...` — отчёты и вывод распознавателя.
