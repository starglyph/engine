# Распознаватель звёздного неба (часть 2)

## Цель

По одному кадру (или короткой серии) оценить **где смотрит камера на небесной сфере** и с каким **полем зрения и ориентацией**, затем наложить **оверлей**: линии созвездий, подписи звёзд/объектов, планеты.

## Декомпозиция задачи

1. **Детекция звёзд** — локальные максимумы после фильтрации фона, или обучаемый детектор.
2. **Фотометрия (грубая)** — аппроксимация блеска для сопоставления с каталогом по относительным величинам.
3. **Сопоставление (star matching)** — найти соответствие между набором (направление, блеск) и подмножеством каталога в предполагаемом поле зрения.
4. **Оценка позы** — RANSAC или robust estimation по парам «звезда на кадре ↔ звезда в каталоге».
5. **Рендер оверлея** — те же проекции, что в симуляторе, но в обратную сторону: небесные линии → пиксели.

Пункты 3–4 часто объединяют в **astrometry**-подобный пайплайн; существуют open-source решения для вдохновения по API и метрикам.

## Источники «истины» для оверлея

- **Созвездия:** линии между звёздами (stick figures) из открытых наборов; границы созвездий IAU — для заливки/подписей регионов.
- **Планеты:** эфемериды на дату съёмки (если дата неизвестна — гипотезы или грубый перебор с оценкой правдоподобия).
- **DSO** (туманности, скопления) — вторичный слой: после стабилизации pose можно подписывать объекты в поле зрения.

## Два пути разработки

### Классический (геометрия + каталог)

Плюсы: интерпретируемость, меньше нужны размеченные реальные фото. Минусы: чувствительность к шуму, облакам, сильной засветке, нестандартной оптике.

### Обучаемый (нейросеть)

Варианты: детектор звёзд + классический matching; end-to-end предсказание параметров камеры; гибрид. Плюсы: устойчивость к артефактам. Минусы: нужны данные (здесь помогает **симулятор**).

## Метрики качества

- **Угловая ошибка** оси взгляда (градусы).
- **Ошибка roll** и FOV (если оцениваются).
- **Доля правильно сопоставленных звёзд** при известной истине (на синтетике).
- **Визуально** — совпадение линий созвездий с реальным небом на контрольном наборе реальных фото.

## Зависимость от симулятора

На синтетических кадрах с известным `meta.json` можно:

- строить кривые обучения;
- ловить систематические ошибки (например, перепутанный полюс);
- регрессионно проверять релизы распознавателя.

## Открытые вопросы

- Нужна ли **калибровка** под конкретное устройство (EXIF + профиль объектива).
- Работа **без даты/места** — сильно шире пространство поиска по эфемеридам планет.
- **Ночной режим** телефонов (стекинг, HDR) — отдельный класс искажений.
